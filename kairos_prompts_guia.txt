INTRODUCCION
============

Este documento contiene los prompts organizados por fases para construir paso
a paso con Claude Code una webapp completa de control horario institucional.
Cada fase es un bloque autocontenido que puede copiarse y ejecutarse
secuencialmente.

NOTA: Antes de cada fase, verificar que la anterior compile y funcione
correctamente. Claude Code trabaja mejor con contexto incremental.

Convenciones:
- Los campos de las tablas de BD usan nombres en espanol excepto los campos de created_at y updated_at.
- Los nombres de las tablas se escribirán en esapñol excepto las tablas y campos originales de la instalación de Laravel 12
- Las FK siguen el patron id_nombre_tabla.
- Los dias habiles son lunes a viernes salvo feriados marcados en calendario.

Estructura:
- Se utiliza la estructura estandar de Laravel 12 sin modificaciones de
  directorios.
- cada migracion debe crear una única tabla correspendiente a su nombre

FASE 14 - Modulo de Marca Web
================================================================================

Implementar el modulo de marcas de E/S desde computadores autorizados,
verificando la identidad del equipo.

--------------------------------------------------------------------------------
PROMPT 14.1 - Modulo de marca desde navegador
--------------------------------------------------------------------------------

Crea el modulo para registrar marcas de E/S desde
computadores autorizados.

TABLA "computadores_autorizados":
- id, id_dispositivo (FK)
- fingerprint (string, unique) -- fingerprint del navegador
- nombre_equipo (string)
- id_dependencia (FK)
- autorizado (boolean)
- creado_en, actualizado_en

FLUJO:
1. Al acceder al modulo, verificar que el computador esta
   autorizado (por fingerprint del navegador)
2. Si no esta autorizado, mostrar pantalla de solicitud
   de autorizacion con datos del equipo
3. Si esta autorizado:
   - Mostrar reloj grande en tiempo real
   - Boton "ENTRADA" (verde) y "SALIDA" (rojo)
   - El usuario se identifica con su documento
   - Al marcar, muestra confirmacion con hora registrada
   - Animacion y feedback visual claro

PANTALLA DE MARCA:
- Fondo con gradiente azul oscuro
- Reloj digital grande centrado (HH:MM:SS)
- Fecha actual debajo del reloj
- Campo de documento con autofocus
- Botones grandes E/S con iconos
- Feedback: "Marca registrada: [ENTRADA/SALIDA] a las HH:MM"
- Logo de la institucion
- Historial de ultimas 5 marcas del equipo

SEGURIDAD:
- Rate limiting: max 1 marca por usuario cada 5 minutos
- Log de intentos fallidos
- Solo accesible desde IPs de la red institucional (configurable)

Crea las rutas, controlador y vista standalone
(no usa el layout principal, tiene su propio diseno).
